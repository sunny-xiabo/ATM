# 需求文档目录

## 目录说明

这是AI测试用例生成系统的需求文档目录。

## 使用说明

1. **放置需求文档**：将PDF、Word等格式的需求文档放在此目录
2. **文档命名**：建议使用清晰的命名，如 `功能需求说明.pdf`、`接口文档.docx` 等
3. **文件格式**：支持PDF、DOC、DOCX等常见格式

## 注意事项

- 大文件（PDF、DOC等）不会被Git跟踪，避免仓库过大
- 小文件（如README、说明文档）会被Git跟踪
- 建议将重要的需求文档放在此目录，便于项目管理和团队协作
